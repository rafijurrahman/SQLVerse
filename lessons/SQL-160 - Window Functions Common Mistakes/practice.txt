Lesson: SQL-160 — Window Functions Common Mistakes
================================

Practice Questions

Question 1:
Common mistake: trying to use a window function alias in WHERE. This is wrong:
SELECT *, RANK() OVER (ORDER BY revenue DESC) AS rnk FROM sales WHERE rnk <= 3.
Write the CORRECT version using a subquery or CTE.

Question 2:
Mistake: LAST_VALUE without correct ROWS frame gives wrong results.
Write the WRONG version: LAST_VALUE(revenue) OVER (ORDER BY sale_date).
Then write the CORRECT version with ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING.

Question 3:
A developer wrote a window function inside WHERE to filter rows. Show the error and the correct pattern:
Instead of WHERE SUM(revenue) OVER () > 5000, use a CTE or subquery wrapper:
SELECT * FROM (SELECT *, SUM(revenue) OVER () AS grand_total FROM sales) AS sub WHERE grand_total > 5000.

Question 4:
Write a corrected version of a query that incorrectly tries to mix window functions with GROUP BY without a CTE:
The wrong version: SELECT category, SUM(revenue), RANK() OVER (ORDER BY SUM(revenue) DESC) FROM sales GROUP BY category.
Fix using: WITH cte AS (...) SELECT ..., RANK() OVER ... FROM cte.
