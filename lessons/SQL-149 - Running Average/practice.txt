Lesson: SQL-149 — Running Average
================================

Practice Questions

Question 1:
Write a query to calculate a running (cumulative) average of revenue:
AVG(revenue) OVER (ORDER BY sale_date) AS running_avg.

Question 2:
Write a query for running average per category:
AVG(revenue) OVER (PARTITION BY category ORDER BY sale_date) AS category_running_avg.

Question 3:
The analyst wants to track whether each sale is above or below the running average at that point in time.
Display: sale_date, product, revenue, AVG(revenue) OVER (ORDER BY sale_date) as "Running Avg",
CASE WHEN revenue > AVG(revenue) OVER (ORDER BY sale_date) THEN 'Above Avg' ELSE 'Below Avg' END as "Status".

Question 4:
Write a query to show the running average of revenue per region, and flag sales that are above their region's running average:
AVG(revenue) OVER (PARTITION BY region ORDER BY sale_date) AS regional_running_avg.
Display: region, sale_date, revenue, running_avg, and above/below flag.
